# План перехода на ialexey.com и интеграции ИИ-стандартов

Этот документ описывает шаги по созданию англоязычной версии сайта на домене **ialexey.com** с полной поддержкой ИИ-агентов на базе Cloudflare для достижения оценки **100/100** на `isitagentready.com`.

---

## Шаг 1. Регистрация домена и настройка Cloudflare DNS

1. **Регистрация домена**: Зарегистрировать домен `ialexey.com`.
2. **Делегирование на Cloudflare**: Направить NS-серверы домена на Cloudflare.
3. **Включение DNSSEC**: Включить DNSSEC в панели Cloudflare и у регистратора для обеспечения cryptographic trust (это проверяется в DNS-AID).
4. **Создание DNS-AID записей**:
   В панели DNS Cloudflare добавить следующие записи типа **HTTPS** (Service Binding):
   
   *   **Запись 1**:
       *   **Тип**: `HTTPS`
       *   **Имя**: `_index._agents`
       *   **Target**: `ialexey.com`
       *   **Параметры/Значение**: `1 . alpn="h2,http/1.1"`
   *   **Запись 2**:
       *   **Тип**: `HTTPS`
       *   **Имя**: `_mcp._agents`
       *   **Target**: `ialexey.com`
       *   **Параметры/Значение**: `1 . alpn="h2,http/1.1"`
   *   **Запись 3**:
       *   **Тип**: `HTTPS`
       *   **Имя**: `_a2a._agents`
       *   **Target**: `ialexey.com`
       *   **Параметры/Значение**: `1 . alpn="h2,http/1.1"`

## Шаг 2. Локализация контента на английский язык

Для полноценного взаимодействия с глобальными ИИ-агентами контент сайта должен быть переведен на английский язык:

1. **Интернационализация (i18n) в Astro**:
   * Настроить поддержку двух языков (мультиязычность `ru`/`en` в Astro) либо сделать отдельную английскую сборку для `ialexey.com`.
2. **Перевод ключевых страниц и файлов**:
   * Главная страница `index.astro` (англоязычные тексты).
   * Лента постов (перевод постов или отдельный фид).
   * Служебные файлы:
     * `llms.txt` (описание на английском языке).
     * `auth.md` (инструкции на английском языке).
     * `SKILL.md` (навык чтения фида на английском).

## Шаг 3. SEO & AEO Контент-стратегия (на базе SEO-материалов)

При наполнении сайта `ialexey.com` использовать лучшие практики AEO (Answer Engine Optimization) и шаблоны блога:

1. **Модель контентных кластеров (Topic Clusters)**:
   * **Главная страница (`ialexey.com` / `/index.md`)** выступает как **Pillar Page** (статья-хаб).
   * **Новости и статьи** выступают как **Cluster Pages** (дочерние статьи). Каждая из них должна ссылаться на главную с нужным анкором (ключевым словом), а главная — линковать эти посты для создания генеалогической иерархии тем.
2. **Метод «Перевернутой пирамиды» для ИИ-сводных ответов (AEO)**:
   * Формулировать частые вопросы в заголовках `H2` (например, *"What is Gemini 1.5 Pro update?"*).
   * Прямой, фактологический ответ в 1–2 предложениях давать **сразу в первом абзаце** под заголовком `H2`. ИИ-боты (Perplexity, SearchGPT) считывают такие структуры в первую очередь.
3. **Использование шаблонов блога (`LaunchPike Templates`)**:
   * Использовать **Template 10 (News)** для быстрых заметок: заголовок с primary keyword, лид-абзац, ключевые факты буллитами, детали и вывод.
   * Использовать **Template 3 (In-depth comparison)** для детальных ИИ-сравнений с использованием HTML-таблиц `<table>`, которые отлично подходят для ИИ-сниппетов.
4. **Техническое SEO и разметка**:
   * Добавлять блок **FAQ (ЧАВО)** в конце статей и размечать его через структурированные данные `FAQPage`.
   * Следить за плотностью ключевых слов (1-2%) и оптимизировать названия картинок (например, `gemini_update_hero.png`) и `alt`-теги.

## Шаг 4. Отслеживание видимости в ИИ-выдаче (AI Visibility Tracker)

Для оценки эффективности оптимизации запустить еженедельный мониторинг по методологии `AI Visibility Training Guide`:

1. **Список целевых ИИ-запросов**:
   Сформировать список из 5–10 поисковых запросов на английском языке, например:
   * *"where to find latest midjourney leaks"*
   * *"best AI news telegram channel 2026"*
   * *"latest updates on GPT-5"*
2. **Проведение еженедельного аудита**:
   * Раз в неделю использовать антидетект-браузер с прокси целевых стран (США, Великобритания, Европа) в инкогнито-режиме.
   * Отправлять запросы в ChatGPT, Gemini и Perplexity.
3. **Фиксация результатов в трекере**:
   Вести таблицу (например, `AI Visibility Tracker.xlsx`), записывая:
   * Страну/прокси и используемый запрос (промпт).
   * Появился ли сайт `ialexey.com` в ответах ИИ в Топ-3 (Да/Нет).
   * Точную позицию ссылки или упоминания (1, 2, 3...).
   * Цитируемые ИИ источники (для анализа конкурентов).

## Шаг 5. Деплой и настройка веб-сервера / правил Cloudflare

На новом домене применить уже готовую структуру служебных файлов из папки `.well-known`:
- `api-catalog`
- `openid-configuration`
- `oauth-authorization-server`
- `oauth-protected-resource`
- `mcp/server-card.json`
- `agent-skills/index.json`

Настроить заголовки и правила Markdown-согласования через Cloudflare Transform Rules или Nginx (как это уже сделано для `.ru`).
